# Glossia Mobile (Expo)

This app uses:
- Expo + React Native
- Native stack navigation (`@react-navigation/native-stack`)
- OAuth 2.0 Authorization Code + PKCE against Glossia

## Environment

Set these environment variables (optional defaults shown):

```bash
EXPO_PUBLIC_GLOSSIA_BASE_URL=https://glossia.ai
EXPO_PUBLIC_GLOSSIA_OAUTH_CLIENT_ID=6b7a2d2b-5f2a-4a57-9ea5-31a4d5b0c5f9
EXPO_PUBLIC_GLOSSIA_OAUTH_SCOPES="account:read project:read"
```

Notes:
- The app defaults to production (`https://glossia.ai`) if `EXPO_PUBLIC_GLOSSIA_BASE_URL` is not set.
- In development builds, use **Change backend URL** on the login screen to switch to local URLs.
- Redirect URI is generated by Expo (`AuthSession.makeRedirectUri`) and shown on the login screen.
- Supported first-party redirect URIs include `glossia://oauth/callback` and Expo Go localhost callbacks (`exp://localhost:8081/--/oauth/callback`, `exp://127.0.0.1:8081/--/oauth/callback`, plus `:19000` variants).
- In development, the login settings modal also lets you override language (`en`, `es`) for quick QA.
- Suggested local URLs:
  - iOS simulator: `http://127.0.0.1:4050`
  - Android emulator: `http://10.0.2.2:4050`
  - Physical device on LAN: `http://<your-local-ip>:4050`

## Run

```bash
npm install
npm run ios
# or
npm run android
```

## Theme

Semantic color tokens in `src/theme/tokens.ts` mirror web semantic tokens from:

`app/priv/static/assets/styles.css`

Use `appTheme.colors.*` in screens/components so mobile and web stay aligned.

## Localization

Localization is configured with:
- `expo-localization` for device locale detection
- `i18n-js` for translation lookup/interpolation
- `src/i18n/I18nContext.tsx` provider + `useI18n()` hook
- `src/i18n/translations.ts` dictionaries (`en`, `es`)

When adding UI copy, use `const { t } = useI18n()` and translation keys instead of hardcoded strings.
